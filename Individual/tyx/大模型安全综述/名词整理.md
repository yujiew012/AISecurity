## 一、核心技术基础类
| 专用名词 | 英文全称 | 中文释义 | 应用场景 |
| --- | --- | --- | --- |
| LLM | Large Language Model | 大语言模型 | 基于海量文本数据预训练，具备自然语言理解与生成能力的深度学习模型，是大模型安全研究的核心对象（如GPT、ChatGLM系列）。 |
| AIGC | Artificial Intelligence Generated Content | 人工智能生成内容 | 利用AI模型（含大语言模型、多模态模型）自动生成文本、图像、音频等内容的技术，其安全风险（如深度伪造、虚假信息）是研究重点。 |
| Transformer | - | Transformer架构 | 基于自注意力机制的深度学习架构，是主流大模型的核心基础，其并行计算能力和长序列处理特性也带来了独特的安全隐患（如注意力机制被对抗攻击利用）。 |
| 多模态大模型 | Multimodal Large Model | - | 融合文本、图像、语音等多种模态数据的大模型（如GPT-4、VisCPM），因跨模态语义鸿沟和数据复杂性，衍生出多模态对抗、跨模态提示注入等安全问题。 |
| 预训练 | Pre-training | - | 大模型训练的第一阶段，在海量无标注数据上学习通用知识与语言规律，此阶段的训练数据质量（如偏见、污染）直接决定模型安全基线。 |
| 微调 | Fine-tuning | - | 基于预训练模型，使用特定任务数据集进一步训练，使模型适配具体场景，微调过程可能被植入后门或引入数据投毒风险。 |
| RLHF | Reinforcement Learning from Human Feedback | 人类反馈强化学习 | 通过人类对模型输出的评分的反馈信号优化模型，是提升模型对齐性的核心技术，但存在反馈数据被污染、偏见放大等安全风险。 |
| LoRA | Low-Rank Adaptation | 低秩适应 | 高效微调技术，通过冻结预训练模型参数、训练低秩矩阵实现模型适配，在隐私保护与模型安全（如防止微调阶段后门注入）中应用广泛。 |
| 思维链 | Chain of Thought, CoT | - | 引导模型逐步推理的提示技术（如“分步解决这个问题”），可提升模型推理能力，但也可能被用于复杂越狱攻击，诱导模型突破安全限制。 |

## 二、安全威胁与攻击类
| 专用名词 | 英文全称 | 核心特点 |
| --- | --- | --- |
| 提示注入攻击 | Prompt Injection Attack | 攻击者通过构造恶意提示词（直接或间接），绕开模型安全规则，诱导模型生成有害内容或执行未授权操作，是最常见的大模型攻击方式。 |
| 越狱攻击 | Jailbreak Attack | 进阶的提示攻击，通过复合指令、密码编码、多模态扰动等方式突破模型安全护栏，使模型脱离预设约束（如伪装学术场景诱导生成犯罪方法）。 |
| 数据投毒攻击 | Data Poisoning Attac | 攻击者在训练数据或用户反馈数据中植入恶意样本、错误标签或后门触发器，导致模型性能下降、决策偏差或触发预设恶意行为。 |
| 后门攻击 | Backdoor Attack | 通过数据投毒或模型修改植入隐藏功能，当输入包含特定触发器（如关键词、图像水印）时，模型输出错误结果，无触发器时正常运行。 |
| 对抗攻击 | Adversarial Attack | 在输入中添加人类难以感知的微小扰动（文本特殊字符、图像像素变化），导致模型输出错误结果，对抗样本具有高度可迁移性。 |
| 模型窃取攻击 | Model Stealing Attack | 攻击者通过API查询、梯度分析等方式，复刻目标模型的功能与性能，或提取模型参数、训练数据分布，侵犯知识产权与核心技术。 |
| 成员推理攻击 | Membership Inference Attack | 攻击者判断某条敏感数据是否属于模型的训练集，可能导致个人隐私（如病历、财务数据）泄露。 |
| 模型逆向攻击 | Model Reverse Attack | 通过模型输出、梯度信息反推模型架构、训练数据特征或核心参数，属于深度模型窃取行为。 |
| 梯度攻击 | Gradient Attack | 白盒场景下，攻击者利用模型梯度信息执行恶意操作（如提取训练数据、植入后门），对开源模型威胁显著。 |
| 自动化攻击 | Automated Attack | 利用自动化工具（如AutoDAN、GPTFuzz）生成攻击提示、迭代优化攻击策略，实现高效、规模化的大模型攻击。 |
| 显存泄漏 | Video Memory Leak | 多租户GPU环境下，模型运行时的显存数据未完全清理，导致跨进程数据窃取，泄露敏感信息或训练数据片段。 |
| 深度伪造 | Deepfake | 利用大模型生成逼真的文本、语音、图像、视频等虚假内容，用于诈骗、造谣等非法活动，是典型的伦理与安全风险。 |

## 三、安全防护与测评类
| 专用名词 | 英文全称 | 核心作用 |
| --- | --- | --- |
| 对抗训练 | Adversarial Training | 将对抗样本融入训练过程，让模型学习对抗特征，提升对扰动输入的鲁棒性，是防御对抗攻击的核心技术。 |
| 模型蒸馏 | Model Distillation | 将大模型的知识迁移到小规模模型中，用小模型对外提供服务，降低大模型被窃取、攻击的风险，同时节省算力。 |
| 差分隐私 | Differential Privacy | 通过在训练数据或模型输出中添加噪声，保护训练数据的隐私，防止成员推理攻击和敏感信息泄露。 |
| 联邦学习 | Federated Learning | 多参与方在不共享原始数据的前提下联合训练模型，实现数据“可用不可见”，从源头降低数据泄露风险。 |
| 数据隔离 | Data Isolation | 通过物理或逻辑手段将模型与敏感数据分离（如模型仅访问数据脱敏后的特征），防止数据直接泄露。 |
| 后门防御 | Backdoor Defense | 包括后门检测（识别训练数据或模型中的后门）与后门移除（如SANDE框架的OSFT技术），保障模型输出的可靠性。 |
| 输出过滤 | Output Filtering | 对模型生成的内容进行后处理，筛选并移除有害信息、敏感数据（如正则表达式匹配银行卡号），是末端安全防护手段。 |
| 水印技术 | Watermarking Technology | 在模型生成内容中嵌入隐形标识（如特定字符组合、语义特征），用于AI生成内容溯源、知识产权保护（如EmbMarker、PromptCARE）。 |
| 红队测试 | Red-teaming | 模拟攻击者视角对模型进行全面安全测试，主动发现模型漏洞（如提示注入、越狱风险），是大模型上线前的关键安全评估环节。 |
| 鲁棒性测评 | Robustness Evaluation | 评估模型在面对输入扰动、对抗攻击、分布外数据时的稳定性，核心测评基准包括AdvGLUE、PromptBench。 |
| 公平性测评 | Fairness Evaluation | 检测模型是否存在性别、种族、宗教等维度的歧视性输出，核心工具与数据集包括GFair、BBQ测评集。 |
| 可解释性测评 | Interpretability Evaluation | 揭示模型决策逻辑（如CritiqueLLM、LLMCheckup），用于定位模型偏见、幻觉生成的根源，辅助安全漏洞排查。 |
