# 核心文件流程梳理
数据预处理（prepare.py）→ 配置覆盖（configurator.py）→ 模型定义（model.py）→ 训练/评估（train.py/bench.py）→ 采样推理（sample.py）

## prepare.py
将原始文本转为模型可直接读取的二进制编码文件，完成「文本→Token 序列→二进制文件」的转换。

## configurator.py
统一管理全流程参数，支持「配置文件+命令行参数」灵活覆盖全局变量，保证参数一致性。

## model.py
实现 GPT 完整结构，支持预训练权重加载，核心包含前向传播和自回归生成两大流程：

### 1. 前向传播（forward）
```plaintext
输入token序列(idx) → 词嵌入(wte) + 位置嵌入(wpe) → Dropout → N个Block依次处理 → 最终层归一化(ln_f) → 输出头(lm_head)生成logits → 若传入targets则计算交叉熵损失
```
- 训练时：输出全序列 logits，计算整体损失（ignore_index=-1）；
- 推理时：仅计算最后一个 token 的 logits，优化计算效率。

### 2. 生成（generate，自回归推理）
```plaintext
初始化序列(idx) → 循环max_new_tokens次：
  裁剪序列到block_size → 前向传播取最后一个token的logits → 温度缩放 → 可选top-k截断 → softmax生成概率 → 采样下一个token → 拼接至原序列 → 最终返回完整生成序列
```

## train.py/bench.py
### train.py（核心训练脚本）

  ```
  加载batch数据（x,y） → 前向传播（混合精度） → 梯度累积 → 梯度裁剪 → 优化器更新 → 日志记录
  ```
### bench.py（性能基准测试）
```
配置初始化 → 模型编译（可选torch.compile） → 预热迭代 → 正式迭代（计算耗时/MFU） → 可选Profiler性能分析
```

## sample.py
基于训练好的模型 / 预训练权重，完成「Prompt→Token→生成文本」的自回归推理。
```plaintext
配置加载+模型初始化 → Prompt编码（字符级/BPE级） → 自回归生成 → Token序列解码为文本 → 输出结果
```
